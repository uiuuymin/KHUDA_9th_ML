KREAM의 거래 데이터에는 명시적인 프리미엄 등급 라벨이 없습니다. 그래서 직접 만들어야 합니다.

01 — K-MEANS CLUSTERING

타겟 라벨을
만드는 이유

프리미엄 비율(premium_ratio)은 연속형 숫자입니다. 이것을 분류 문제로 바꾸려면 "몇 등급"인지 라벨이 필요합니다.

🤔
왜 KMeans인가?
premium_ratio 분포를 보면 자연스럽게 여러 덩어리(cluster)가 형성됩니다. KMeans는 이 덩어리를 수학적으로 찾아줍니다. 사람이 임의로 "0~20%는 낮음" 같이 경계를 정하면 주관이 개입되지만, KMeans는 데이터 자체의 패턴을 따릅니다.
premium_ratio → log 변환 → StandardScaler → KMeans(K=5)
📐
왜 log 변환을 하나?
premium_ratio는 오른쪽으로 긴 꼬리(right-skewed) 분포를 가집니다. 극단적인 폭등 거래가 소수 있어서, 그대로 쓰면 KMeans가 이상치에 끌려갑니다. log1p 변환으로 분포를 정규화해 클러스터가 더 균형 있게 나뉩니다.
np.log1p(premium_ratio.clip(lower=-0.99))
최적 K 탐색: Elbow + Silhouette
두 지표를 함께 사용한 이유:
Elbow는 언제 성능 향상이 줄어드는지를,
Silhouette는 클러스터가 실제로 잘 나뉘는지를 측정합니다.
▲ Elbow: K=4 이후 감소폭이 급격히 줄어듦

02 — DATA SPLIT STRATEGY

왜 일반 random split을
쓰지 않나?

데이터 누수(Data Leakage)를 막는 것이 이 프로젝트에서 가장 중요한 설계 결정입니다.

02
GroupShuffleSplit
같은 상품(product_id)의 거래 기록이 train과 test에 동시에 들어가면, 모델이 "이 신발의 과거 패턴"을 외워버립니다. 실제 서비스에서는 새 상품에 대한 예측이 필요하므로 반드시 제품 단위로 분리합니다.
3가지 누수 방지 장치
  • GroupShuffleSplit(group=product_id) — 동일 제품이 train/test에 걸치지 않도록 제품 단위 분리
  • KMeans는 train으로만 fit — test에는 predict만 적용. train의 클러스터 경계가 test에 "새나가지" 않음
  • 라벨 순서 매핑도 train 기준 고정 — "낮음=0, 높음=4" 정의가 train에서 한 번만 결정되고 test에 그대로 적용
  • TH4 튜닝도 train 내부 val에서만 — test를 한 번도 보지 않은 상태에서 TH4 결정
gss = GroupShuffleSplit(n_splits=1, test_size=0.2, random_state=42) train_idx, test_idx = next(gss.split(df, groups=product_id)) # KMeans: train만 fit kmeans.fit_predict(train_scaled) ← ✅ kmeans.predict(test_scaled) ← ✅ kmeans.fit_predict(all_data) ← ❌ 누수 발생!
쉬운 비유

시험을 앞두고 공부할 때, 시험지를 미리 보고 공부하면 안 됩니다. 일반 random split은 마치 시험지를 뒤섞어서 일부를 공부 자료에 넣는 것과 같습니다. GroupShuffleSplit은 "이 선생님이 낸 문제 스타일은 배웠지만, 이 특정 시험지는 처음 봤다"는 상황을 만들어줍니다.

04 — TH4 POST-PROCESSING

소수 클래스를
더 잘 잡는 방법

Label 4(폭등)는 데이터가 극히 적어 기본 예측에서 Recall이 낮습니다. TH4 후처리로 이를 보완합니다.

TH4 아이디어

XGBoost는 예측 시 각 클래스에 대한 확률값을 반환합니다. 기본적으로 가장 높은 확률의 클래스를 예측(argmax)하지만,

"Label 4일 확률이 TH4% 이상이면, 다른 클래스 확률이 더 높더라도 Label 4로 예측"

이렇게 하면 Label 4를 더 많이 잡을 수 있습니다. 단, 일부 오분류도 늘어나므로 Precision과 Recall의 Trade-off가 발생합니다.

TH4 작동 원리 — 임계값 위치에 따른 예측 변화
TH4를 낮게 설정하면: Label 4 Recall ↑, Precision ↓
TH4를 높게 설정하면: Label 4 Recall ↓, Precision ↑
TH4=0.10
최적 TH4
P(label=4) = 0.05 0.25 0.50
TH4 튜닝이 테스트셋 누수가 아닌 이유
  • TH4 탐색은 train 내부 val에서만 수행
  • val도 GroupShuffleSplit으로 제품 단위 분리
  • test는 TH4 결정 과정에서 한 번도 참조되지 않음
  • 결정된 TH4를 test에 적용 → 공정한 최종 평가
TH4 적용 전후 Label 4 성능 변화 (개념적)

Precision은 낮아지지만 Recall이 3배 향상됩니다. "폭등 신발을 놓치는 것"이 더 큰 손해이므로 이 Trade-off는 비즈니스 맥락에서 합리적입니다.
05 — FEATURE SELECTION

VIF로
다중공선성 제거

서로 비슷한 정보를 담고 있는 피처들을 동시에 넣으면 모델이 혼란스러워집니다. VIF로 이를 측정하고 정리합니다.

📏
VIF란? (분산팽창지수)
한 피처가 다른 피처들로 얼마나 설명되는지를 측정합니다. VIF = 1이면 완전 독립, VIF ≥ 10이면 다른 피처와 강하게 상관되어 있어 모델에서 중복 정보를 제공하는 것입니다.
VIF < 5 → 안전 VIF 5~10 → 주의 VIF ≥ 10 → 제거 ← 이 프로젝트 기준
🎨
왜 color_* 피처는 VIF 적용 안 하나?
원핫 인코딩(One-Hot Encoding)된 컬럼들은 구조적으로 VIF가 높게 나옵니다. color_white=1이면 나머지 색상은 자동으로 0이 되므로 "인공적인" 상관관계가 생깁니다. 이는 다중공선성이 아니라 인코딩의 특성이므로 통째로 포함합니다.

"왜 Accuracy가 아닌 F1-Macro를 쓰나?" — 클래스 불균형 상황에서 Accuracy는 다수 클래스만 맞춰도 높게 나옵니다. F1-Macro는 Label 0~4 각각의 F1을 동등하게 평균내므로, 희귀한 Label 4를 얼마나 잘 맞추는지도 점수에 반영됩니다.

07 — PRESENTATION GUIDE

발표 핵심 포인트
& 예상 질문

교수님이 주목할 가능성이 높은 포인트와 예상 질문 답변입니다. 발표 자료 슬라이드 구성에도 참고하세요.

핵심 설계 결정
왜 KMeans + XGBoost 조합인가?
명시적 라벨이 없는 문제를 비지도→지도 2단계로 해결한 것이 이 프로젝트의 핵심 아이디어입니다.
Q. 처음부터 회귀로 하면 안 됐나요?
A. 가능하지만 "어느 정도 오차가 의미 있는가"의 기준이 없습니다. 5구간 분류로 만들면 사용자에게 명확한 등급 정보를 줄 수 있습니다.
데이터 무결성
데이터 누수를 어떻게 막았나?
GroupShuffleSplit + KMeans train-only fit + TH4 val-only 튜닝. 3중 누수 방지 구조입니다.
Q. 일반 train_test_split과 뭐가 다른가요?
A. 같은 제품의 거래가 train/test에 걸치면 모델이 "해당 제품의 패턴"을 외웁니다. 제품 단위 분리로 진짜 일반화 성능을 측정합니다.
소수 클래스 전략
Label 4 대응: 2가지 방어선
1차: sample_weight로 학습 시 Label 4에 높은 가중치. 2차: TH4 후처리로 예측 단계에서 추가 보완.
Q. Precision이 낮아지는 건 문제 아닌가요?
A. 비즈니스 맥락에서 "폭등 신발을 놓치는 것(FN)"이 "잘못 예측하는 것(FP)"보다 손해가 큽니다. 의도적인 Trade-off입니다.
모델 해석성
Feature Importance로 설명 가능성 확보
recent_avg_premium, days_since_release가 상위. 과거 프리미엄 이력과 출시 후 경과일이 가장 중요한 신호입니다.
Q. VIF 기준 10은 왜인가요?
A. 통계학의 일반적 기준입니다(Kutner et al., 2004). VIF=10은 해당 피처의 분산 중 90%가 다른 피처로 설명된다는 의미입니다.
평가 전략
왜 F1-Macro를 주 지표로 쓰나?
클래스 불균형 시 Accuracy는 다수 클래스만 잘 맞춰도 높게 나옵니다. F1-Macro는 모든 클래스를 동등하게 평가합니다.
Q. F1-Weighted와 다른가요?
A. Weighted는 샘플 수 비례 평균, Macro는 단순 평균입니다. Label 4처럼 적은 클래스를 동등하게 보려면 Macro가 더 적합합니다.

Q. 하이퍼파라미터를 왜 그냥 고정했나요?
A. 이번은 파이프라인 검증이 목적입니다. n_estimators=500, lr=0.05, max_depth=6은 XGBoost 안정적 기본 조합이며, 최적화는 다음 단계 과제입니다.

팀원을 위한 용어 사전
premium_ratio
(거래가 - 정가) / 정가. 0.5면 정가보다 50% 비싸게 팔린 것
Inertia (관성)
클러스터 내부 점들이 중심에서 얼마나 퍼져 있는지. 낮을수록 촘촘한 클러스터
Silhouette Score
클러스터가 얼마나 잘 분리됐는지. -1~1, 1에 가까울수록 좋음
Data Leakage
테스트 정보가 학습에 흘러들어가 성능이 부풀려지는 현상
VIF (분산팽창지수)
피처 간 선형 중복 정도. ≥10이면 다중공선성 심각, 제거 권고
F1-Macro
각 클래스의 F1 점수를 단순 평균. 소수 클래스도 동등하게 반영
Feature Importance (Gain)
각 피처가 트리 분기에서 얼마나 성능 향상에 기여했는지
TH4 후처리
예측 단계에서 Label 4 확률이 임계값 이상이면 강제 4 예측. Recall 향상 목적
